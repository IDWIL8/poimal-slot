# Поймал слот — поймал коуча

Сайт для бронирования постоянного времени индивидуальной встречи с Ингой на весь квартал.

- Неделя 1 — встречи во время планирования спринта.
- Неделя 2 — встречи во время спринт-ревью.
- Один человек может выбрать только один слот.
- Встреча длится 45 минут, но в календаре резервируется один час.
- Имена записавшихся видны всем, чтобы коллеги могли договориться об обмене.
- Управлять своей записью можно по личной ссылке без входа в аккаунт.
- Административная панель доступна только через разрешённый Google-аккаунт.

Ниже — подробная инструкция для человека, который раньше не запускал сайты и не работал с Firebase.

## Что понадобится

Перед началом подготовьте:

1. Компьютер с macOS, Windows или Linux.
2. Аккаунт Google — для Firebase и входа администратора.
3. Аккаунт [GitHub](https://github.com/) — чтобы сохранить код.
4. Аккаунт [Netlify](https://www.netlify.com/) — чтобы опубликовать сайт.
5. Около 30–45 минут свободного времени.

Платная подписка для первого запуска не нужна.

## Шаг 1. Установите Node.js

Node.js нужен для запуска и сборки сайта.

1. Откройте [nodejs.org](https://nodejs.org/).
2. Скачайте версию с пометкой **LTS**.
3. Запустите установщик и соглашайтесь с рекомендуемыми настройками.
4. После установки откройте приложение **Терминал**.

На macOS Терминал находится в **Программы → Утилиты → Терминал**.

Вставьте команду:

```bash
node --version
```

Если появился номер вроде `v20.11.0` или выше — всё готово.

Затем проверьте npm:

```bash
npm --version
```

Если обе команды показывают номера версий, переходите дальше.

## Шаг 2. Откройте папку проекта

В Терминале выполните:

```bash
cd "/Users/tkachukinga/Documents/PO 1-1"
```

Кавычки нужны, потому что в названии папки есть пробел.

Проверьте, что вы находитесь в правильной папке:

```bash
pwd
```

В ответе должен быть путь, заканчивающийся на `PO 1-1`.

## Шаг 3. Установите зависимости

Зависимости — это готовые библиотеки, которые использует сайт.

Выполните:

```bash
npm install
```

Команда может работать несколько минут. Предупреждения жёлтого цвета обычно не мешают запуску. Важно, чтобы в конце не было сообщения `npm error`.

## Шаг 4. Посмотрите демонстрационную версию

Запустите сайт:

```bash
npm run dev
```

Терминал покажет адрес, например:

```text
Local: http://127.0.0.1:5173/
```

Откройте именно тот адрес, который показан в Терминале. Если порт `5173` занят другим сайтом, Vite автоматически выберет `5174`, `4173` или другой свободный порт.

Пока Firebase не настроен, сверху будет надпись **«Демо-режим»**. В этом режиме:

- можно проверить внешний вид и бронирование;
- данные сохраняются только в текущем браузере;
- другие пользователи не увидят эти записи;
- после очистки данных браузера записи исчезнут.

Чтобы остановить сайт, вернитесь в Терминал и нажмите `Control + C`.

## Шаг 5. Создайте проект Firebase

Firebase будет хранить реальные бронирования и обновлять расписание у всех пользователей.

1. Откройте [Firebase Console](https://console.firebase.google.com/).
2. Нажмите **Create a project** или **Создать проект**.
3. Введите название, например `inga-coaching-booking`.
4. Google Analytics можно отключить — для этого сайта он не обязателен.
5. Нажмите **Create project**.
6. Дождитесь завершения создания и нажмите **Continue**.

### Добавьте веб-приложение

1. На главной странице проекта нажмите значок `</>` — **Web**.
2. Введите название приложения, например `booking-web`.
3. Галочку **Firebase Hosting** ставить не нужно: сайт будет опубликован на Netlify.
4. Нажмите **Register app**.
5. Firebase покажет блок `firebaseConfig`.

Он будет выглядеть примерно так:

```js
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  storageBucket: "your-project.firebasestorage.app",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

Не закрывайте эту страницу — значения понадобятся на следующем шаге.

## Шаг 6. Создайте базу Firestore

1. В левом меню Firebase откройте **Build → Firestore Database**.
2. Нажмите **Create database**.
3. Выберите **Start in production mode**.
4. Выберите ближайший регион. После создания базы изменить регион будет сложно.
5. Нажмите **Enable**.

Коллекции вручную создавать не нужно. Сайт создаст их после первого бронирования.

## Шаг 7. Включите вход через Google

Google-вход используется только для административной панели.

1. В Firebase откройте **Build → Authentication**.
2. Нажмите **Get started**.
3. Откройте вкладку **Sign-in method**.
4. Выберите **Google**.
5. Включите переключатель **Enable**.

7. Нажмите **Save**.

## Шаг 8. Заполните файл `.env.local`

Вернитесь в Терминал, откройте папку проекта и выполните:

```bash
cp .env.example .env.local
```

На macOS файл можно открыть в стандартном текстовом редакторе командой:

```bash
open -e .env.local
```

Заполните файл значениями из `firebaseConfig`:

```env
VITE_FIREBASE_API_KEY=значение_apiKey
VITE_FIREBASE_AUTH_DOMAIN=значение_authDomain
VITE_FIREBASE_PROJECT_ID=значение_projectId
VITE_FIREBASE_STORAGE_BUCKET=значение_storageBucket
VITE_FIREBASE_MESSAGING_SENDER_ID=значение_messagingSenderId
VITE_FIREBASE_APP_ID=значение_appId
VITE_ADMIN_EMAIL=ваш_google_email@gmail.com
VITE_SPRINT_START=2026-07-06
```

Соответствие полей:

| Firebase | `.env.local` |
|---|---|
| `apiKey` | `VITE_FIREBASE_API_KEY` |
| `authDomain` | `VITE_FIREBASE_AUTH_DOMAIN` |
| `projectId` | `VITE_FIREBASE_PROJECT_ID` |
| `storageBucket` | `VITE_FIREBASE_STORAGE_BUCKET` |
| `messagingSenderId` | `VITE_FIREBASE_MESSAGING_SENDER_ID` |
| `appId` | `VITE_FIREBASE_APP_ID` |

Важно:

- не добавляйте кавычки вокруг значений;
- не ставьте пробелы до или после `=`;
- `VITE_ADMIN_EMAIL` должен совпадать с Google-аккаунтом администратора;
- `VITE_SPRINT_START` должен быть понедельником первой недели в формате `ГГГГ-ММ-ДД`;
- файл `.env.local` нельзя публиковать в GitHub — он уже добавлен в `.gitignore`.

Сохраните файл и полностью перезапустите сайт:

```bash
npm run dev
```

Надпись **«Демо-режим»** должна исчезнуть.

## Шаг 9. Укажите email администратора в правилах

Откройте файл `firestore.rules` в редакторе кода или TextEdit.

Найдите строку:

```text
ADMIN_EMAIL_REPLACE_ME
```

Замените её на тот же адрес, который указан в `VITE_ADMIN_EMAIL`. Например:

```text
inga@example.com
```

Email в `.env.local` и `firestore.rules` должен совпадать символ в символ.

## Шаг 10. Опубликуйте правила Firestore

В Терминале, находясь в папке проекта, выполните:

```bash
npx firebase-tools login
```

Откроется браузер. Войдите в тот Google-аккаунт, на котором создан Firebase-проект, и разрешите доступ.

Затем выполните:

```bash
npx firebase-tools init firestore
```

Firebase задаст несколько вопросов:

1. **Please select an option** — выберите **Use an existing project**.
2. Выберите созданный Firebase-проект.
3. **What file should be used for Firestore Rules?** — укажите `firestore.rules`.
4. Если появится вопрос о перезаписи существующего файла — ответьте **No**.
5. Для файла индексов можно оставить предложенное имя `firestore.indexes.json`.

Теперь опубликуйте правила:

```bash
npx firebase-tools deploy --only firestore:rules
```

Успешный результат заканчивается сообщением вроде `Deploy complete!`.

Без этого шага реальные бронирования могут не сохраняться.

## Шаг 11. Проверьте Firebase локально

1. Запустите сайт командой `npm run dev`.
2. Откройте адрес из Терминала.
3. Забронируйте свободный тестовый слот.
4. Скопируйте личную ссылку управления.
5. Откройте Firebase Console → **Firestore Database → Data**.

После бронирования должны появиться коллекции:

- `slotClaims` — занятые слоты и публичные имена участников;
- `bookingIdentities` — защита от повторной записи;
- `bookings` — полные данные бронирований;
- `adminNotes` — приватные заметки администратора.

Дополнительная проверка:

1. Откройте сайт в обычном окне браузера.
2. Откройте тот же адрес в режиме инкогнито.
3. Забронируйте слот в одном окне.
4. Во втором окне слот должен почти сразу стать занятым и показать имя.

После проверки удалите тестовую запись через личную ссылку или административную панель.

## Шаг 12. Проверьте административную панель

1. Откройте адрес сайта и добавьте `/admin` в конец.

Пример:

```text
http://127.0.0.1:5173/admin
```

2. Нажмите **Войти через Google**.
3. Выберите аккаунт из `VITE_ADMIN_EMAIL`.

В административной панели можно:

- просматривать все записи;
- искать и сортировать участников;
- переносить и удалять встречи;
- редактировать имя, команду и тему;
- вести приватные заметки;
- отмечать проведённые встречи;
- смотреть статистику;
- выгружать таблицу CSV.

Если появляется сообщение, что доступа нет, ещё раз сравните email в `.env.local`, `firestore.rules` и выбранном Google-аккаунте.

## Шаг 15. Опубликуйте сайт на Netlify

1. Откройте [Netlify](https://app.netlify.com/) и войдите через GitHub.
2. Нажмите **Add new project → Import an existing project**.
3. Выберите GitHub.
4. Разрешите Netlify доступ к репозиторию `coaching-booking`.
5. Выберите этот репозиторий.

Настройки сборки уже находятся в `netlify.toml`:

```text
Build command: npm run build
Publish directory: dist
```

Пока не публикуйте сайт — сначала добавьте переменные окружения.

### Добавьте переменные в Netlify

1. Откройте **Site configuration → Environment variables**.
2. Нажмите **Add a variable**.
3. По очереди добавьте все значения из `.env.local`:
   - `VITE_FIREBASE_API_KEY`;
   - `VITE_FIREBASE_AUTH_DOMAIN`;
   - `VITE_FIREBASE_PROJECT_ID`;
   - `VITE_FIREBASE_STORAGE_BUCKET`;
   - `VITE_FIREBASE_MESSAGING_SENDER_ID`;
   - `VITE_FIREBASE_APP_ID`;
   - `VITE_ADMIN_EMAIL`;
   - `VITE_SPRINT_START`.
4. Сохраните изменения.
5. Откройте **Deploys** и нажмите **Trigger deploy → Deploy site**.

Через несколько минут Netlify покажет адрес примерно такого вида:

```text
https://your-site-name.netlify.app
```

## Шаг 16. Разрешите домен Netlify в Firebase

Без этого Google-вход администратора на опубликованном сайте может не работать.

1. Скопируйте адрес сайта Netlify без `https://` и без завершающего `/`.

Пример:

```text
your-site-name.netlify.app
```

2. Откройте Firebase Console.
3. Перейдите в **Authentication → Settings → Authorized domains**.
4. Нажмите **Add domain**.
5. Вставьте домен Netlify.
6. Сохраните.

Теперь откройте опубликованный сайт и проверьте `/admin` ещё раз.

Перед тем как отправить ссылку коллегам, проверьте:

- быть понедельником;

После изменения локально перезапустите `npm run dev`. Для опубликованного сайта измените переменную в Netlify и запустите новый deploy.

## Частые проблемы

### Открылся другой сайт

Посмотрите адрес, который показала команда `npm run dev`. Если `5173` занят, сайт запускается на другом порту. Откройте новый адрес целиком.

### Страница не открывается

Убедитесь, что Терминал с `npm run dev` всё ещё работает. Если Терминал закрыт, запустите команду снова.

### Всё ещё отображается демо-режим

Проверьте:

- файл называется именно `.env.local`, а не `.env.local.txt`;
- все значения Firebase заполнены;
- после изменения файла сервер был остановлен и запущен снова.

### Ошибка `Missing or insufficient permissions`

Обычно это означает, что правила Firestore не опубликованы. Выполните:

```bash
npx firebase-tools deploy --only firestore:rules
```

Если сайт показывает сообщение **«Firebase отклонил запись»**, сначала выполните ту же команду, дождитесь `Deploy complete!`, а затем обновите страницу сочетанием `Command + Shift + R`.

### Не работает вход администратора

Проверьте четыре пункта:

1. Google включён в Firebase Authentication.
2. Email одинаковый в `.env.local` и `firestore.rules`.
3. Выбран правильный Google-аккаунт.
4. Домен Netlify добавлен в **Authorized domains**.

### После изменений видна старая версия

На macOS нажмите `Command + Shift + R`. На Windows — `Control + F5`. Если это Netlify, сначала убедитесь, что новый deploy завершён успешно.

### Порт уже занят

Это не ошибка. Vite выберет следующий свободный порт. Используйте адрес из строки `Local:`.

## Важно о личных ссылках

Личная ссылка вида `/manage/случайный-токен` работает как ключ. Любой человек, получивший эту ссылку, сможет изменить или отменить соответствующую запись. Не публикуйте такие ссылки в общих чатах.
